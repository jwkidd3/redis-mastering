What is Redis?

Redis = REmote DIctionary Server
  • In-memory data structure store - All data stored in RAM for ultra-fast access
  • NoSQL key-value database - Simple, flexible data model
  • Multi-purpose platform - Database, cache, message broker, streaming engine
  • Open-source - BSD licensed, widely adopted
  • Single-threaded - Atomic operations, consistent data

Single-threaded Architecture Benefits

Redis uses a single-threaded event loop for command processing

Insurance Transaction Benefits:

  • Atomic operations - No race conditions in premium calculations
  • Predictable performance - Consistent response times for quotes
  • Simple debugging - Easier to troubleshoot claim processing
  • No locks needed - Eliminates deadlock scenarios

Client-Server Communication

RESP Protocol Introduction

RESP = REdis Serialization Protocol
  • Binary-safe protocol - Handle policy documents safely
  • Simple text format - Easy debugging and monitoring
  • Command pipelining - Batch insurance operations
  • Pub/Sub support - Real-time claim notifications
  • Cross-platform - Works on Windows and Mac
