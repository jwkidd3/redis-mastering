What is Redis?

Redis = REmote DIctionary Server

  • Open-source, in-memory data structure store
  • Used as database, cache, and message broker
  • Supports multiple data structures
  • Network accessible via TCP protocol

Redis Key Characteristics

  • In-Memory: All data stored in RAM for speed
  • Persistent: Optional data durability to disk
  • Single-Threaded: Atomic operations, no race conditions
  • Network Protocol: Client-server architecture
  • Data Structures: Rich set of data types

Redis vs Traditional SQL Databases

Aspect Redis SQL Database
Storage In-Memory Disk-based
Speed Sub-millisecond Milliseconds+
Schema Schema-less Fixed Schema
Query Language Commands SQL

Redis Architecture Overview

Single-Threaded Event Loop

Client → Network → Event Loop → Memory → Response

  • Single process handles all operations
  • Event-driven, non-blocking I/O
  • Commands processed sequentially
  • No concurrency issues within Redis

Memory Management

All data stored in RAM for maximum performance

  • Volatile Memory: Data lost on restart (default)
  • Persistence Options: RDB snapshots, AOF logs
  • Memory Policies: Eviction when memory full
  • Compression: Automatic data structure optimization

Redis Data Structures Preview

  • Strings: Text, numbers, binary data
  • Hashes: Field-value pairs (like objects)
  • Lists: Ordered sequences
  • Sets: Unique values
  • Sorted Sets: Scored unique values
  • Streams: Append-only logs

Performance Characteristics

Typical Performance: 100,000+ operations/second

  • Read Operations: O(1) for most commands
  • Write Operations: Atomic and immediate
  • Network Latency: Main bottleneck
  • Memory Access: Direct RAM access

Redis in Application Architecture

Typical Architecture

Application ← → Redis ← → Database

Cache Layer: Reduces database load
Session Store: User session management
Queue: Background job processing

Connection & Configuration

Default Configuration

  • Port: 6379
  • Host: Remote server (not localhost)
  • Database: 16 databases (0-15), default is 0
  • Authentication: Optional password

Course Environment

# Connection details will be provided redis-cli -h [remote-host] -p 6379

Security Basics

Important: Redis is designed for trusted environments

  • Network Security: Firewall protection
  • Authentication: Password protection
  • Authorization: Command restrictions
  • Encryption: SSL/TLS for data in transit

Why Redis is So Fast

  • In-Memory Storage: No disk I/O for reads
  • Simple Data Structures: Optimized algorithms
  • Single-Threaded: No context switching
  • Efficient Protocols: Minimal overhead
  • Pipelining: Batch command execution

Result: Sub-millisecond response times
