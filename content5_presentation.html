Redis Sets: Unique Collections

🔑 Definition: Redis Sets are unordered collections of unique string members, perfect for storing distinct values and performing set arithmetic operations.

📊 Key Characteristics

  • Unique members: No duplicates allowed
  • Unordered: No specific sequence
  • Fast membership: O(1) checking
  • Set arithmetic: Union, intersection, difference
  • Memory efficient: Optimized storage

🎯 Common Use Cases

  • User segmentation: Group users by behavior
  • Tag systems: Product categories, user interests
  • Access control: Permission groups, roles
  • Unique tracking: Visitors, sessions, events
  • Feature flags: A/B testing groups
# Basic set operations
SADD users:premium "user001" "user003" "user007"
SADD users:active "user001" "user002" "user003" "user004"
SADD users:mobile "user002" "user003" "user005"

# Check membership
SISMEMBER users:premium "user001"  # Returns 1 (true)

# Get all members
SMEMBERS users:premium

Set Arithmetic Operations

Intersection (AND)

# Premium users who are also active
SINTER users:premium users:active
# Returns: user001, user003

Union (OR)

# All premium OR mobile users
SUNION users:premium users:mobile
# Returns all unique users from both sets

Difference (NOT)

# Active users who are NOT premium
SDIFF users:active users:premium
# Returns: user002, user004

Store Results

# Save intersection result
SINTERSTORE users:premium_active 
  users:premium users:active

Redis Sorted Sets: Ranked Collections

🔑 Definition: Redis Sorted Sets are collections of unique members, each associated with a score. Members are automatically sorted by their scores.

📊 Key Features

  • Scored members: Each member has a numeric score
  • Auto-sorted: Ordered by score (ascending)
  • Unique members: No duplicates
  • Range queries: By score or rank
  • Fast updates: O(log N) operations

🎯 Perfect For

  • Leaderboards: User rankings, scores
  • Priority queues: Task scheduling
  • Time series: Timestamp-based data
  • Analytics: Performance metrics
  • Rate limiting: Request counting
# Basic sorted set operations
ZADD user_scores 850 "user001" 720 "user002" 950 "user003"
ZADD sales_leaderboard 15000 "agent001" 23000 "agent002" 8500 "agent003"

# Get top performers (highest scores first)
ZREVRANGE sales_leaderboard 0 2 WITHSCORES

# Get user's rank
ZREVRANK sales_leaderboard "agent002"

Sorted Set Range Operations

Score-Based Queries

# Get users with scores between 800-900
ZRANGEBYSCORE user_scores 800 900 WITHSCORES

# Get high performers (score > 20000)
ZRANGEBYSCORE sales_leaderboard 20000 +inf WITHSCORES

# Count users in score range
ZCOUNT user_scores 700 850

# Get users with top 10% scores
ZREVRANGEBYSCORE user_scores +inf -inf LIMIT 0 10

Rank-Based Operations

# Get top 5 performers
ZREVRANGE sales_leaderboard 0 4 WITHSCORES

# Get bottom 3 performers
ZRANGE sales_leaderboard 0 2 WITHSCORES

# Get user's specific score
ZSCORE sales_leaderboard "agent001"

# Update scores
ZINCRBY sales_leaderboard 5000 "agent001"

User Activity Scoring

# JavaScript consumer aggregating user activity from events
const updateUserActivity = async (event) => {
  const userId = event.userId;
  
  switch (event.eventType) {
    case 'user.login':
      await redis.zIncrBy('activity_scores', 1, userId);
      break;
    case 'order.completed':
      await redis.zIncrBy('activity_scores', 10, userId);
      await redis.zIncrBy('revenue_leaderboard', event.amount, userId);
      break;
    case 'product.viewed':
      await redis.zIncrBy('activity_scores', 0.5, userId);
      break;
    case 'review.submitted':
      await redis.zIncrBy('activity_scores', 5, userId);
      break;
  }
};

# Composite scoring (weighted average)
// JavaScript function to calculate composite scores
const updateCompositeScore = async (agentId) => {
  const orders = await redis.zScore('performance:orders', agentId) || 0;
  const revenue = await redis.zScore('performance:revenue', agentId) || 0;
  const satisfaction = await redis.zScore('performance:satisfaction', agentId) || 0;
  
  // Weighted composite score
  const composite = (orders * 0.3) + (revenue * 0.0001) + (satisfaction * 20);
  await redis.zAdd('performance:composite', { score: composite, value: agentId });
};

Performance & Best Practices

Set Operations

  • SADD, SREM: O(1)
  • SISMEMBER: O(1)
  • SMEMBERS: O(N)
  • SINTER, SUNION: O(N×M)

Sorted Set Operations

  • ZADD, ZREM: O(log N)
  • ZSCORE, ZRANK: O(log N)
  • ZRANGE: O(log N + M)
  • ZRANGEBYSCORE: O(log N + M)
⚠️ Best Practices:
  • Use SSCAN instead of SMEMBERS for large sets
  • Limit range queries with LIMIT parameter
  • Consider memory usage for large sorted sets
  • Use appropriate data types for scores (integers when possible)
  • Implement pagination for large result sets

Memory Optimization

# Efficient operations for large datasets
SSCAN users:active 0 MATCH "user*" COUNT 100  # Paginated scanning
ZRANGEBYSCORE leaderboard 0 1000 LIMIT 0 20   # Limited range queries
ZUNIONSTORE temp_result 2 set1 set2 WEIGHTS 1 2  # Weighted unions

// Complex segmentation logic
const analyzeUser = async (userId, orderEvent) => {
  // Add to active buyers
  await redis.sAdd('segment:active_buyers', userId);
  
  // High-value customer check
  if (orderEvent.amount > 1000) {
    await redis.sAdd('segment:high_value', userId);
  }
  
  // Get user's current segments
  const segments = await Promise.all([
    redis.sIsMember('segment:premium', userId),
    redis.sIsMember('segment:high_value', userId),
    redis.sIsMember('segment:mobile', userId)
  ]);
  
  return {
    isPremium: segments[0],
    isHighValue: segments[1],
    isMobile: segments[2]
  };
};

Sorted Set Analytics

// Leaderboard management
const updateLeaderboard = async (userId, score) => {
  await redis.zAdd('user_leaderboard', { score, value: userId });
};

// Get user rankings
const getUserRanking = async (userId) => {
  const score = await redis.zScore('user_leaderboard', userId);
  const rank = await redis.zRevRank('user_leaderboard', userId);
  const total = await redis.zCard('user_leaderboard');
  
  return {
    score,
    rank: rank + 1, // Convert to 1-based ranking
    total,
    percentile: ((total - rank) / total * 100).toFixed(1)
  };
};
